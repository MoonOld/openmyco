# OpenMyco - 产品规格说明

## 项目概述

**OpenMyco** 是一个通过 LLM 问答进行知识学习的结构化平台。用户输入知识点，LLM 分析返回相关知识体系，以可视化知识图谱展示，支持递归扩展。

## 核心功能

### 1. 知识图谱生成
- 用户输入一个知识点（如 "React Hooks"）
- LLM 自动生成相关知识图谱，包括：
  - 当前知识点的详细描述
  - 前置知识（学习前需要掌握的）
  - 后置知识（学习后可以继续学习的）
  - 相关知识（有关联的内容）

### 2. 可视化展示
- 以图（节点 + 边）的形式展示知识关系
- 不同关系类型用不同颜色区分：
  - 前置知识 - 红色
  - 后置知识 - 绿色
  - 相关知识 - 蓝色
  - 依赖关系 - 紫色
  - 包含关系（contains）- 橙色
- 节点展示：标题、描述、类型、难度等级

### 3. 交互式探索
- 点击节点可查看详细信息
- 节点可展开/折叠，递归扩展相关知识
- 支持拖拽、缩放、平移图谱
- **聚焦模式**：选中节点时只展示该节点及其邻居（1-3度）
- **全局视图**：展示所有节点
- **节点编辑**：点击节点底部"编辑"按钮可修改标题、描述、类型、难度等
- **重新布局**：一键重新组织图谱布局

### 3.2 Q&A 互动探索
- 选中节点后切换到"问答"Tab 可对节点提问
- LLM 回答后提供 4 种沉淀动作供用户选择：
  - **仅保存**：保存问答记录，不修改节点内容
  - **合并到字段**：将回答合并到节点的 principle/useCases/bestPractices/commonMistakes 字段
  - **生成子话题**：将问答生成一个新的 subTopic
  - **升级为节点**：将问答升级为一个独立的知识节点，并创建关联边
- 问答历史持久化，随节点数据导出/导入
- 支持传入已有 QA 历史摘要，防止 LLM 重复回答

### 3.1 分层加载体验
- 用户点击"探索"后立即显示骨架节点（只有标题）
- 骨架节点显示"正在加载详情..."状态
- 深度信息加载后自动填充内容
- 加载中的节点禁用"探索"按钮

### 4. 本地持久化
- 所有知识图谱保存在本地 IndexedDB
- 支持历史记录查看
- 支持导出/导入 JSON 格式
- **图谱管理**：
  - 侧边栏显示所有历史图谱
  - 支持新建图谱（清空当前图谱）
  - 支持删除单个图谱
  - 支持清除所有数据
  - 每次输入新知识点会创建独立的新图谱

## 非功能需求

| 需求 | 说明 |
|------|------|
| 纯本地应用 | 无需后端服务，数据仅存储在本地 |
| LLM 兼容性 | 支持 OpenAI Compatible API |
| 离线可用 | 生成的知识图谱可离线查看 |
| 隐私保护 | 知识图谱不离开用户设备 |

## 用户使用流程

```
1. 配置 API Key (设置)
   ↓
2. 输入知识点 (聊天界面)
   ↓
3. LLM 生成知识图谱
   ↓
4. 查看可视化图谱
   ↓
5. 点击节点扩展更多知识
   ↓
6. 对节点提问，选择沉淀动作（问答 Tab）
   ↓
7. 本地保存，随时查看
```

## 技术约束

- 前端框架：React 19 + TypeScript
- 构建工具：Vite
- 图可视化：React Flow
- 状态管理：Zustand
- 本地存储：IndexedDB (Dexie.js)
- UI 组件：shadcn/ui + Tailwind CSS
- 桌面框架：Electron（可选）

## 平台支持

### Web 版本
- 浏览器运行，无需安装
- 访问地址：`http://localhost:5173`（开发）或部署服务器
- 支持所有现代浏览器

### 桌面版本（Electron）
- 一键安装，本地运行
- 支持 Windows、macOS、Linux
- 打包后体积约 150-200MB
- 开箱即用，无需浏览器环境
